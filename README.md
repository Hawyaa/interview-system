# 🎯 ProctorAI — AI-Powered Proctored Interview System

A complete, production-ready interview platform built with Next.js 14, Supabase, Groq AI, face-api.js, and the Web Speech API.

---

## 📁 Project Structure

```
/app
  page.js                        → Redirects to /login
  /login/page.js                 → HR login & signup
  /dashboard/page.js             → HR dashboard (create sessions, view results)
  /interview/[token]/page.js     → Candidate interview page (main feature)
  /report/[id]/page.js           → HR report viewer
  /api/interview-turn/route.js   → Server-side Groq API proxy

/components
  FaceMonitor.js                 → Webcam face detection (face-api.js)
  SpeechInput.js                 → Voice + text answer input
  ProctorWarning.js              → Floating warning banner for flags

/lib
  supabase.js                    → Supabase client
  groq.js                        → Groq API helper

/public/models/                  → face-api.js model weights (download below)
```

## 🚀 Step-by-Step Deployment Guide

### STEP 1 — Get your free accounts

1. **Supabase** (database + auth): https://supabase.com → Create free project
2. **Groq** (AI): https://console.groq.com → Sign up → API Keys → Create key
3. **Vercel** (hosting): https://vercel.com → Sign up with GitHub

---

### STEP 2 — Set up Supabase

1. In Supabase Dashboard, go to **SQL Editor**
2. Click **New Query**
3. Paste the entire contents of `supabase-schema.sql` and click **Run**
4. Go to **Project Settings → API**
5. Copy:
   - **Project URL** → `NEXT_PUBLIC_SUPABASE_URL`
   - **anon public key** → `NEXT_PUBLIC_SUPABASE_ANON_KEY`

---

### STEP 3 — Download face-api.js model files

The face detection models must be in `/public/models/`. Run these commands:

```bash
mkdir -p public/models

# Tiny face detector
curl -L -o public/models/tiny_face_detector_model-weights_manifest.json \
  https://raw.githubusercontent.com/justadudewhohacks/face-api.js/master/weights/tiny_face_detector_model-weights_manifest.json

curl -L -o public/models/tiny_face_detector_model-shard1 \
  https://raw.githubusercontent.com/justadudewhohacks/face-api.js/master/weights/tiny_face_detector_model-shard1

# Face landmark 68 tiny
curl -L -o public/models/face_landmark_68_tiny_model-weights_manifest.json \
  https://raw.githubusercontent.com/justadudewhohacks/face-api.js/master/weights/face_landmark_68_tiny_model-weights_manifest.json

curl -L -o public/models/face_landmark_68_tiny_model-shard1 \
  https://raw.githubusercontent.com/justadudewhohacks/face-api.js/master/weights/face_landmark_68_tiny_model-shard1
```

---

### STEP 4 — Configure environment variables

1. Copy `.env.local` (already created) and fill in your real values:

```env
NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_ID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=YOUR_ANON_KEY
GROQ_API_KEY=gsk_YOUR_GROQ_KEY
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

---

### STEP 5 — Install and run locally

```bash
npm install
npm run dev
```

Open http://localhost:3000 → redirects to /login → create an HR account.

---

### STEP 6 — Push to GitHub

```bash
git init
git add .
git commit -m "Initial ProctorAI commit"
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git push -u origin main
```

---

### STEP 7 — Deploy on Vercel (free)

1. Go to https://vercel.com → **Add New Project**
2. Import your GitHub repository
3. In **Environment Variables**, add all four variables from Step 4
   - Change `NEXT_PUBLIC_SITE_URL` to your Vercel URL (e.g. `https://proctoral.vercel.app`)
4. Click **Deploy**
5. Done! Share the `/interview/[token]` links with candidates.

## 🔄 How the Interview Flow Works

```
HR creates session → token generated → share link sent to candidate
Candidate opens link → grants camera, mic, screen share
Interview begins → Groq asks Q1 (voice + text)
Candidate answers (voice or type) → API route calls Groq → scored
Loop until all N questions answered
Final summary generated by Groq → saved to Supabase
HR views report at /report/[id]
```

## 🔒 Proctoring Flags Explained

| Flag Type           | Trigger                          |
|---------------------|----------------------------------|
| `TAB_SWITCH`        | `visibilitychange` event fires   |
| `NO_FACE`           | face-api.js detects 0 faces      |
| `MULTIPLE_FACES`    | face-api.js detects 2+ faces     |
| `SCREEN_SHARE_STOPPED` | Candidate ends screen share   |
| `WRONG_SCREEN_SHARE`| Shared window instead of monitor |

All flags are timestamped and stored in `interview_results.flags_json`.
